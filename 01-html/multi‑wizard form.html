Multi-Wizard Form Guide

What is it? A "Multi-Wizard Form" is simply a long form broken down into small, manageable steps (pages). Think of a checkout process: Shipping → Billing → Review.

The browser sees it as one single form, but we use CSS and JavaScript to show only one section at a time to the user.

How to Build It (The Logic)

1 The Structure

Wrap everything in one <form> tag. Create individual steps using <section> or <div> wrappers. 

<form action="/submit" method="POST">

  <!-- Step 1 -->
  <section id="step1">
    <h2>Account Info</h2>
    <input type="text" name="username" />
    <button type="button" onclick="nextStep(2)">Next</button>
  </section>

  <!-- Step 2 -->
  <section id="step2" style="display:none;">
    <h2>Personal Details</h2>
    <input type="email" name="email" />
    <button type="submit">Finish</button>
  </section>

</form>

2 Key Inputs & Attributes

Always use the name attribute. Without it, the server won't receive that field's data. 

<!-- CORRECT -->
<input type="text" name="full_name" />

<!-- WRONG (Server sees nothing) -->
<input type="text" />

Deep Dive Q&A

  • 1. What is the difference between "Save" and "Next" buttons? Next/Prev Buttons: Usually <button type="button">. They don't submit the form; they just trigger JavaScript to change the visible section.

    Submit Button: <button type="submit">. Found on the last step, it sends ALL the data from ALL steps to the server at once.
  • 2. Why use one single <form> tag? If you used separate forms for each step, you'd have to save data to a database or local storage after every single click. Using one big form keeps all the data on the page until the very end, which is simpler for basic wizards.
  • 3. How do you handle validation in a wizard? You typically check validation before moving to the next step. If Step 1 has an error, you don't hide Step 1 to show Step 2; you show the error message instead.
    JS Concept 
    function goToNext() {
  if (input.value === "") {
    alert("Please fill this out!");
    return; // Don't move forward
  }
  showNextStep(); 
}
  • 4. Core Form Elements Refresher <label>: Connects text to inputs (click text to focus input).
    <input type="email">: Validates formatting and shows @ keyboard on mobile.
    <textarea>: For multi-line text (address, bios).

Server Side Notes

  • How does the server receive the data? When you click "Finish" (Submit), the browser bundles up every field with a name attribute from Step 1, Step 2, etc., and sends them together. The server receives one big object: { username: "...", email: "..." }.
