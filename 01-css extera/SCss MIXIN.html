CSS Custom Functions & Mixins

Introduction: The CSS Custom Functions and Mixins module is an exciting upcoming feature that brings programming-like capabilities directly to native CSS. Historically, developers relied on preprocessors like SASS or LESS to use functions and mixins. This module aims to make those features native to the browser, allowing for dynamic value calculation and reusable style patterns without a build step.

Concept Questions & Examples

  • 1. What is the CSS custom functions and mixins module? It is a CSS module that lets developers create reusable blocks of CSS that can accept arguments, contain complex logic (using things like if() and @media), and return values. This brings power previously consistent with preprocessors directly into the browser.
    Example Scenario

    Instead of repeating box-shadow: 0 4px 6px rgba(0,0,0,0.1) everywhere, you could define it once as a reusable mixin and apply it to any card or button.

  • 2. What are CSS custom functions and how are they defined? CSS custom functions are functions you define in CSS using the @function at‑rule. They can be called as values inside any CSS property and return a value based on their arguments and internal logic.
    Code Example 
    @function --double(--val) {
  result: calc(var(--val) * 2);
}
  • 3. What does <dashed-function> syntax look like? It looks similar to custom properties, but with parentheses and arguments, for example --my-function(30px, 3). This <dashed-function> form is how you call a custom function in property values.
    Code Example 
    .box {
  /* Calling a custom function */
  width: --double(50px); /* Becomes 100px */
}
  • 4. What are CSS mixins in this module? CSS mixins are reusable sets of CSS properties defined using @mixin and applied using @apply inside rulesets. They can be customized with arguments and logic so you can reuse patterns of styles across multiple selectors.
    Code Example 
    @mixin --center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

.hero {
  @apply --center-flex; /* Applies the styles above */
  height: 100vh;
}
  • 5. Can custom functions and mixins have typed arguments and return values? Yes, they can be given optional data types for their arguments and their return values. This restricts what values can be passed in and what can be returned, making them safer and more predictable.
    Code Example 
    /* Function only accepts 'length' (px, rem, etc.) */
@function --spacer(type(length) --size) {
  result: calc(var(--size) * 1.5);
}

Support / Status Questions

  • 6. Which of these features are currently supported in browsers? Currently, only CSS custom functions have partial browser support (mainly in experimental builds). CSS mixins are not supported in any browser yet.
    Note

    You usually need to enable "Experimental Web Platform features" in Chrome flags to test these today.

  • 7. Which at‑rules does this module introduce that are not yet supported? Besides @function, the module introduces @mixin, @apply, @contents, and @env. At the moment, no browsers support these additional at‑rules.

Reference and API Questions

  • 8. Which at‑rule is central to CSS custom functions, and what descriptor is associated with it? The central at‑rule is @function. It has a result descriptor that represents the value returned by the function.
    Syntax Refresher 
    @function --name {
  result: cyan; /* 'result' is the return statement */
}
  • 9. Name the main interfaces associated with CSS custom functions. The key interfaces are CSSFunctionRule, CSSFunctionDeclarations, and CSSFunctionDescriptors. These are used in the CSSOM to represent and work with custom functions in JavaScript.

Relationship / Context Questions

  • 10. How do CSS custom functions relate to existing CSS math and value functions like calc() or clamp()? They live in the same “values and units” space as functions like calc(), max(), min(), clamp(), and trigonometric or math functions. You can combine custom functions with these built‑in functions to build more powerful and reusable value logic.
    Code Example 
    width: calc(100% - --sidebar-width()); 
/* Combining built-in calc() with a custom function */
  • 11. Where is the behavior of CSS functions and mixins formally defined? It is defined in the “CSS Functions and Mixins Module” specification. MDN links directly to the current draft of this spec.

“See also” / Navigation

  • 12. Which related CSS features does the page highlight in “See also”? It highlights the if() value function and the @media at‑rule as related features. These are examples of tools you can use for logic inside custom functions or mixins.
    Logic Example 
    @function --text-color(--bg) {
  /* Using if() for logic inside a function */
  result: if(var(--bg) == black, white, black);
}
